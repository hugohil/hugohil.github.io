Le design procédural

Le design procédural, c'est concevoir des règles plutôt que des résultats. Le programme produit la forme, le designer conçoit le programme.

Sol LeWitt ; Proposal for Wall Drawing, Information Show (MoMA, 1970). Le dessin est une suite d'instructions exécutées par d'autres.

10 PRINT CHR$(205.5+RND(1)); : GOTO 10 ; 10print.org. Un programme Commodore 64 d'une seule ligne. Une seule règle simple et de l'aléatoire créent de la complexité visuelle.

0xA ; expr~ - 2011.

Ryoji Ikeda ; test pattern, 2008. Un système qui convertit n'importe quelle donnée (texte, image, son) en motifs de codes-barres (0 et 1).

Matt DesLauriers ; Meridian, 2021, meridian.mattdesl.com. Un programme JavaScript de 15 ko qui construit des reliefs par centaines de milliers de traits colorés.

Les automates cellulaires (cellular automaton)

“une grille régulière de « cellules » contenant chacune un « état » [...] À chaque nouvelle unité de temps, les mêmes règles sont appliquées simultanément à toutes les cellules de la grille, produisant une nouvelle « génération » de cellules dépendant entièrement de la génération précédente.” (Wikipédia)

John Conway's Game of Life (1970)

Le motif du "blinker"

Chaque cellule est vivante ou morte. Elle a 8 voisines. À chaque génération :

  • une cellule vivante avec moins de 2 voisines vivantes meurt (isolement)
  • une cellule vivante avec 2 ou 3 voisines vivantes survit
  • une cellule vivante avec plus de 3 voisines vivantes meurt (surpopulation)
  • une cellule morte avec exactement 3 voisines vivantes devient vivante

4 règles simples créent des comportements impossibles à prévoir.

Random walker #1

Un agent (walker) se déplace sur une grille. L'orchestrateur lance un D6 pour décider du mouvement de l'agent :

  • D6 = 1 ou 3 ou 5 : avancer d'un pas
  • D6 = 2 : rotation +90°
  • D6 = 4 : rotation −90°
  • D6 = 6 : rotation 180°
10 minutes

Random walker #3

Structure d'un programme

function setup() {
  // exécuté UNE SEULE FOIS au démarrage
  createCanvas(400, 400);
  background(220);
}

function draw() {
  // exécuté EN BOUCLE (~60 fois par seconde)
}

setup() = préparer l'espace de travail. draw() = dessiner en continu.

Le système de coordonnées : (0, 0) est en haut à gauche. X augmente vers la droite, Y augmente vers le bas.


  // pour "commenter" une ligne de code : ajouter "//" devant.
  // elle ne sera pas exécutée, c'est juste une note pour le lecteur humain.

Variables

Sans variables, chaque modification impose de changer chaque valeur manuellement :

// difficile à modifier ; 6 valeurs liées
circle(200, 200, 80);
circle(200, 200, 60);
circle(200, 200, 40);

Avec des variables, on peut facilement modifier plusieurs valeurs en une seule fois :

let x = 200;
let y = 200;

circle(x, y, 80);
circle(x, y, 60);
circle(x, y, 40);

Une variable peut aussi changer au fil du temps :

let x = 0;  // déclaré ici : x existe partout, persiste entre les frames

function draw() {
  background(220);
  circle(x, 200, 50);
  x = x + 2;  // prends x, ajoute 2, range le résultat dans x
}

x = x + 2 n'est pas une équation ; c'est une instruction. L'endroit où une variable est déclarée détermine où elle existe : c'est son scope. Déclarée en dehors de draw(), elle persiste d'une frame à l'autre.

p5.js fournit aussi des variables système qu'il met à jour automatiquement :

mouseX, mouseY    // position de la souris ; variables comme les autres
width, height     // dimensions du canvas
frameCount        // images écoulées depuis le début
// ...et beaucoup d'autres

Types et opérations

Une variable peut contenir différents types de valeurs :

let x = 200;          // number ; entier
let vitesse = 0.5;    // number ; décimal
let nom = "Hugo";     // string ; texte entre guillemets
let actif = true;     // boolean ; vrai ou faux (true / false)

Sur les nombres, on va faire des opérations :

x + 2     // addition
x - 2     // soustraction
x * 2     // multiplication
x / 2     // division
x % 3     // modulo

Pour le modulo %, pensez à la monnaie du pain : si vous avez 7€ et qu'un objet coûte 3€, vous achetez 2 objets et il vous reste 1€ ; donc 7 % 3 = 1. C'est particulièrement utile en design génératif : frameCount % 60 donne un compteur qui repart à zéro toutes les 60 frames.

Conditions et boucles

// condition
if (mouseX > width / 2) {
  fill(255, 0, 0);  // souris à droite : rouge
} else {
  fill(0, 0, 255);  // souris à gauche : bleu
}

// boucle
for (let i = 0; i < 5; i++) {
  let x = i * 80;
  circle(x, 200, 40);
}
